# Engineering workflow

## Repository architecture

Grainlab is a Tauri v2 macOS app with no Node.js or frontend bundler.

```text
ui/                         static frontend
  index.html                application shell
  style.css                 layout and visual system
  main.js                   state, interactions, processing fallback, crop, export
  gpu.js                    WebGPU orchestration and uniforms
  grain.js                  CPU grain implementation
  shaders/photo.wgsl        primary GPU photo pipeline
  film-stocks/              source stock definitions and dossiers
  film-stocks/index.json    generated manifest

src-tauri/
  src/lib.rs                native application runner
  build.rs                  stock discovery, validation, manifest generation
  tauri.conf.json           window, capabilities, and bundle configuration

assets/                     source icons, symbols, and README screenshots
scripts/                    setup, checks, development, diagnostics, packaging
dist/                       verified local release application
```

Keep frontend work in `ui/`. Add Rust/Tauri code only when the capability genuinely belongs at the native boundary.

## Generated and source-owned files

- Source stock JSON files under category folders are canonical.
- `ui/film-stocks/index.json` is generated by `src-tauri/build.rs` during Cargo-backed commands.
- `assets/icons/AppIcon-1024.png` is the canonical app icon source.
- The icon source must be exactly 1024 × 1024 RGBA with transparent corners; do not rely on macOS to mask an opaque legacy ICNS. Validate the Dock-sized packaged result.
- `src-tauri/icons/` is generated by `cargo tauri icon`.
- `target/` and `dist/` are build outputs.

Do not manually make a stock exist only in `index.json`. Do not hand-edit platform icons when the source icon should change.

## Development commands

Run commands from the repository root:

```bash
./scripts/setup.sh
./scripts/doctor.sh
./scripts/dev.sh
./scripts/check.sh
./scripts/build_macos_app.sh
```

Equivalent `make` targets exist for setup, doctor, dev, check, visual-test, learn-style, icons, build-app, clean, and help.

`scripts/check.sh` verifies shell, JavaScript, Python, and Swift syntax, the app-icon source contract, Rust formatting, Clippy with warnings denied, and Rust tests. Because the Tauri build script runs during Cargo compilation, it also validates film-stock data and regenerates the manifest.

`scripts/run_visual_tests.sh` is the opt-in renderer-calibration path. It downloads only manifest-pinned, licensed fixtures into the ignored `artifacts/` tree, creates deterministic charts and iPhone QA inputs, runs the grain-model regression, and builds the local report. Baselines must be produced by Grainlab's real GPU or CPU renderer rather than a duplicated test implementation.

`scripts/learn_film_style.sh` is the local personal-style path. It analyzes positive scans without uploading them, writes a candidate stock plus measurement evidence and an HTML report beneath the ignored `artifacts/learned-styles/` tree, and installs only with an explicit flag. `make learn-style INPUT=/path/to/roll ID=my-roll NAME="My Roll"` installs the definition and runs Cargo so the common stock validator rebuilds the manifest. The result must remain labeled as a scan-derived interpretation, not a measured stock characterization.

## macOS packaging

`scripts/build_macos_app.sh` is the canonical release path. It:

1. verifies required macOS tools;
2. reads default name, bundle id, and version from repository configuration;
3. validates optional metadata overrides;
4. regenerates icons when needed;
5. runs a locked Tauri release build;
6. copies the `.app` to `dist/`;
7. verifies metadata, executable, icon, and signature;
8. applies an ad-hoc local signature if required.

Use `UNIVERSAL=1` for an Apple Silicon and Intel universal build. Packaging changes are not complete until the resulting `dist/Grainlab.app` opens and important native behavior is exercised.

## Runtime testing

Static checks cannot prove native drag regions, WebGPU availability, WebKit layout, or packaged asset freshness. For material changes:

- build the release app;
- quit any older running Grainlab bundle so macOS does not reopen stale packaged resources;
- launch the exact `dist/Grainlab.app` path;
- confirm the active engine label;
- exercise the affected UI in the real app;
- capture screenshots when visual judgment matters.

An earlier QA pass accidentally targeted an older already-running `dist` process even while a development binary was being built. Always identify which bundle is actually running before trusting runtime evidence.

## Upstream template

The reusable foundation is configured as:

```text
upstream  git@github.com:adammenges/rust_template.git
```

Grainlab and the template originated with unrelated Git histories. Do not blindly merge the entire upstream branch over the product. Fetch and audit upstream changes, then integrate the best applicable improvements selectively while preserving Grainlab’s UI, film pipeline, capabilities, and product-specific configuration.

Typical review areas include toolchain pins, Tauri configuration, capabilities, CI, scripts, packaging validation, icon generation, and Rust application structure.

## Dirty worktrees and scope

This repository may contain extensive user-owned changes. Before editing:

- inspect `git status --short`;
- do not reset, checkout, delete, or overwrite unrelated changes;
- make the smallest coherent patch within the requested scope;
- treat generated files carefully;
- do not commit or stage unless explicitly requested.

Use `apply_patch` for text edits. Keep shell scripts executable, Bash-compatible, and guarded with `set -euo pipefail`.

## Documentation ownership

- `README.md` is the product, setup, and contributor overview.
- `ui/film-stocks/README.md` is the canonical stock schema guide.
- `.agents/` records durable product and engineering intent for future agents.
- `FEEDBACK.md` records concise lessons from explicit user corrections.
- `design-qa.md` and `artifacts/design-qa/` retain visual comparison evidence.

When implementation changes a documented feature, update the relevant documentation in the same task.

## Final verification checklist

Choose checks proportional to the change, but use the full sequence for renderer, architecture, packaging, or major UI work:

1. `git diff --check`
2. `./scripts/check.sh`
3. `./scripts/build_macos_app.sh`
4. `./scripts/doctor.sh`
5. Packaged-app runtime QA

Also confirm:

- film manifest generation is stable;
- no WGSL compilation fallback occurred unexpectedly;
- canvas dimensions still equal source dimensions;
- README and `.agents` context match behavior;
- the app is left in a clean, understandable state for handoff.
